# AWS Interview Question: Services Used in AWS

## Interview Question

**What AWS services have you used?**

---

# Sample Interview Answer (2–3 Minutes)

> "I have worked with a variety of AWS services across compute, storage, networking, databases, security, monitoring, and DevOps. For compute, I have used EC2, Auto Scaling, Load Balancers, and Lambda. For storage, I have worked with S3, EBS, and EFS. In networking, I have configured VPCs, subnets, route tables, Internet Gateways, NAT Gateways, Security Groups, Network ACLs, Route 53, and VPC Endpoints. For databases, I have used Amazon RDS and Aurora. For security, I have worked with IAM, KMS, Secrets Manager, and AWS Certificate Manager. For monitoring and logging, I have used CloudWatch, CloudTrail, and VPC Flow Logs. For DevOps and automation, I have experience with CloudFormation, CodePipeline, CodeBuild, CodeDeploy, Systems Manager, and Amazon ECR for container images."

# AWS Services by Category

## 1. Compute Services

These services provide computing power to run applications.

| Service | Purpose |
|---------|---------|
| EC2 | Virtual servers |
| Lambda | Serverless computing |
| ECS | Run Docker containers |
| EKS | Managed Kubernetes |
| Elastic Beanstalk | Deploy web applications |
| Auto Scaling | Automatically scale EC2 instances |
| Application Load Balancer (ALB) | Distribute HTTP/HTTPS traffic |
| Network Load Balancer (NLB) | Distribute TCP/UDP traffic |

---

## 2. Storage Services

Used to store application data.

| Service | Purpose |
|---------|---------|
| S3 | Object storage |
| EBS | Block storage for EC2 |
| EFS | Shared file storage |
| S3 Glacier | Long-term archival storage |
| AWS Backup | Centralized backup service |

---

## 3. Networking Services

These services connect and secure AWS resources.

| Service | Purpose |
|---------|---------|
| VPC | Private network |
| Subnets | Divide the VPC |
| Route Tables | Traffic routing |
| Internet Gateway | Internet connectivity |
| NAT Gateway | Outbound internet for private subnets |
| Security Groups | Stateful firewall |
| Network ACLs | Stateless firewall |
| Route 53 | DNS service |
| VPC Endpoints | Private access to AWS services |
| Direct Connect | Dedicated connection to AWS |
| VPN | Secure connection to AWS |

---

## 4. Database Services

Managed database services.

| Service | Purpose |
|---------|---------|
| RDS | Managed relational database |
| Aurora | High-performance relational database |
| DynamoDB | NoSQL database |
| ElastiCache | In-memory caching |
| Redshift | Data warehouse |

---

## 5. Security & Identity Services

Manage access and protect resources.

| Service | Purpose |
|---------|---------|
| IAM | Users, roles, and permissions |
| KMS | Encryption key management |
| Secrets Manager | Store passwords and API keys |
| AWS Certificate Manager (ACM) | SSL/TLS certificates |
| AWS WAF | Web application firewall |
| AWS Shield | DDoS protection |
| Amazon Cognito | User authentication |

---

## 6. Monitoring & Logging

Monitor application health and AWS activity.

| Service | Purpose |
|---------|---------|
| CloudWatch | Metrics, logs, and alarms |
| CloudTrail | API activity auditing |
| AWS Config | Resource configuration tracking |
| VPC Flow Logs | Network traffic logs |
| AWS X-Ray | Application tracing |

---

## 7. DevOps & Automation

Automate deployments and infrastructure.

| Service | Purpose |
|---------|---------|

## 8. Containers

Run containerized applications.

| Service | Purpose |
|---------|---------|

## 9. Messaging & Integration

Enable communication between applications.

| Service | Purpose |
|---------|---------|

---

## 10. Migration & Disaster Recovery

Move workloads to AWS and protect data.

| Service | Purpose |
|---------|---------|
| AWS Database Migration Service (DMS) | Database migration |

| AWS Backup | Backup management |
